## 项目简介

**玄鲸 / Whale-LLT / Whale 小手机** 是一个面向 AI Roleplay 与情感陪伴场景的移动端拟真聊天应用。项目尝试把传统“聊天框式”的 LLM 对话，扩展为更接近手机社交应用的长期陪伴体验：聊天、语音、群聊、朋友圈、论坛、表情包、角色设定、长期记忆、一起听歌等功能被整合在一个移动端友好的界面中。

项目基于 Web 技术开发，并通过 **Capacitor** 打包为 Android 原生应用。它本身**不提供模型服务、不内置后端内容生成能力**，用户需要自行配置第三方模型 API 或本地模型服务。聊天数据、角色数据与大部分配置主要保存在用户本地环境中。

这个项目最初源于一个很简单的想法：

> 想让 AI 不只是“回答问题”，而是更像一个可以被长期陪伴、共同互动、一起生活在手机里的角色。

## 项目亮点

- **移动端拟真体验**：模拟手机聊天与社交应用体验，适合 AI Roleplay 与情感陪伴场景。
- **Capacitor Android 打包**：Web 应用可被转换为 Android App，支持更完整的移动端体验。
- **本地优先**：用户数据主要保存在本地浏览器数据库或 App 本地环境中，降低对中心化服务器的依赖。
- **角色长期记忆**：支持角色信息、对话上下文、全局记忆等能力，增强长期陪伴感。
- **多模态交互探索**：支持语音消息、电话、识图、图片/表情素材等更自然的互动形式。
- **社交场景扩展**：除一对一聊天外，还包含群聊、朋友圈、论坛/微博式互动、一起听歌等模块。
- **手机能力接入**：通过 Capacitor 接入文件读写、文件选择、本地通知、安全区域适配等移动端能力。
- **真实用户反馈驱动**：项目经过长期真实用户使用、反馈、修复与迭代。


## Capacitor 移动端打包指南

本项目使用 **Capacitor** 将 Web 应用转换为 Android 原生应用。以下内容用于说明当前移动端打包相关能力与基本开发流程。

### 已实现的移动端能力

1. **基础 Capacitor 项目设置**
   - 初始化 npm 项目。
   - 安装并配置 Capacitor。
   - 添加 Android 平台。

2. **刘海屏与底部导航栏适配**
   - 配置 Android 应用使用透明状态栏和导航栏。
   - 通过 CSS 实现安全区域适配。

3. **手机存储功能：数据导入 / 导出**
   - 使用 Capacitor Filesystem 插件实现文件写入。
   - 使用 Capacitor FilePicker 插件实现文件选择。
   - 支持在 App 内导出和导入备份 JSON 文件。

4. **本地通知功能**
   - 使用 Capacitor LocalNotifications 插件实现本地通知。
   - 添加 Android 通知权限处理。

5. **App 图标设置指南**
   - 提供 Android Studio 中设置应用图标的详细说明。
   - 详见 `app-icon-guide.md`。

6. **APK 构建打包指南**
   - 提供使用 Android Studio 构建 APK 的步骤说明。
   - 详见 `build-apk-guide.md`。

## 设计理念

### 1. 从“工具”到“陪伴”

很多 LLM 产品默认把用户放在“提问者”的位置，而 Whale 小手机更关注 AI 角色在日常情境中的存在感：消息通知、语音、动态、群聊、表情包、记忆与社交互动，都是为了让角色更像“生活在手机里”。

### 2. 降低使用门槛

项目尽量避免复杂后端部署，让普通用户可以通过静态页面、本地浏览器或 APK 直接体验。对于不熟悉服务器和插件系统的用户，这种方式更轻量。

### 3. 本地优先与隐私意识

项目尽可能把用户数据保存在本地，减少中心化后端依赖。用户仍需自行注意 API 服务、网络代理、模型供应商与设备环境可能带来的数据风险。

### 4. 用户反馈驱动迭代

项目长期来自真实用户反馈：哪些角色更自然，哪些交互更像陪伴，哪些功能会打断沉浸感，哪些移动端细节容易失败，都会反过来影响后续设计。

## 常见问题

### Q：这个项目提供 AI 模型或 API 吗？

不提供。项目本身是前端交互工具，用户需要自行配置模型 API 或本地模型服务。

### Q：聊天数据会上传到服务器吗？

项目设计上以本地存储为主，不提供中心化后端聊天服务。但如果你配置了第三方模型 API、ASR、TTS、代理或图片服务，相关请求可能会发送到对应第三方服务。请自行阅读对应服务商的隐私政策和数据处理规则。

### Q：可以在手机上用吗？

可以。项目支持 Android APK 版本，也可以在移动端浏览器中运行，但不同设备和浏览器的兼容性可能存在差异。

### Q：Web 代码改了，为什么 Android App 里没有变化？

修改 Web 代码后，需要重新同步到 Android 项目：

```bash
npx cap sync
```

然后再在 Android Studio 中重新运行或构建。

### Q：可以二次开发吗？

可以。请遵守本仓库开源许可证、免责声明与相关法律法规，并保留必要的版权与来源说明。

### Q：为什么有些功能不稳定？

当前版本仍处于 Release Beta 阶段，部分功能会受设备、浏览器权限、模型 API、网络环境、本地存储机制和 WebView 行为影响。
